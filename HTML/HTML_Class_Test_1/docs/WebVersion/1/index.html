접근방법

저는 반사라는 개념을 스스로 이해하는 내에서 만들기 위해서 다수의 if문을 사용하여 해결하고자 하였습니다. 노란색 박스(이하 Player)로 다수의 공이 접근해 올 때, 이 공이 어디서 접근하였는지 알 수 있게 하기 위해서 Player 안에 그것을 구분하기 위한 공간을 만들어 해결하고자 하였습니다.

그림으로 표현하면 위와 같습니다.


변수설명

우선 충돌이라는 개념은 isCollision이라는 함수를 통해서 판단할 수 있었지만 이를 반사시키기 위해서는 스스로 이해할 수 있는 코드가 필요하였고 저는 코드를 재구성하였습니다. 실제로 변경된 코드는 3.model.2.moveball.js.bak 뿐 입니다.

각각의 공이 충돌했을때 어느 공간에서 접근했는지 구분하기 위하여 각각의 boolean 변수를 만들었습니다. 기본적으론 false이며 접근했을때 true가 됩니다.

  • var left;
    • 왼쪽에서 접근했는가?
  • var right;
    • 오른쪽에서 접근했는가?
  • var top;
    • 위쪽에서 접근했는가?
  • var bottom;
    • 아래쪽에서 접근했는가?


함수설명

이후 공이 모서리로 접근했으나 정방향에서 맞은 경우가 아닌 것을 감지하여 적절한 방향으로 반사시킵니다.

if(top && keyStat[1]) {

    else {

        this.sx = MeMe.sx;
        this.y = MeMe.y - this.r;

} else if(top) {
    // 아닌 개체는 튕겨나감
    this.y -= this.sy;
    this.sy = this.sy * -1;
}

MeMe가 움직이는 경우라면 공이 충돌후 접근해온 방향으로 MeMe가 공을 밀어주도록 합니다. 공이 더 빠른 경우라면 공의 가속도를 올려주고, MeMe가 더 빠른 경우라면 공이 MeMe와 같은 가속도를 가지게 됩니다.


해결한 과제

  • 공을 알맞은 방향으로 반사시킬 수 있습니다.
  • 공을 알맞은 방향으로 밀어줄 수 있습니다.


해결하지 못한 과제

  • 감지할 수 없는 공간 안으로 접근한 경우 개체가 같혀버리는 상황이 발생합니다.
